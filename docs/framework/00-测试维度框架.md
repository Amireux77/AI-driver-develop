### 1.1.6 约束

**单个约束**

| 维度        | 测试点                                         | 说明                                     |
| ----------- | ---------------------------------------------- | ---------------------------------------- |
| NOT NULL    | 为该类型列添加 NOT NULL 约束                   | 插入 NULL 应报错                         |
| UNIQUE      | 为该类型列添加 UNIQUE 约束                     | 插入重复值应报错                         |
| PRIMARY KEY | 将该类型列作为主键                             | 唯一且非空                               |
| FOREIGN KEY | 将该类型列作为外键，引用另一表主键             | 验证插入/更新/删除时的引用完整性         |
| CHECK       | 为该类型列添加 CHECK 约束 (如 col > 0)         | 插入违反条件的值应报错                   |
| 自增长      | 将该类型列定义为自增长/IDENTITY/AUTO_INCREMENT | 验证插入时自动生成值、显式插入行为及边界 |
| 复合约束    | 多列组合 UNIQUE 约束 (包含该类型列)            | 组合值唯一                               |
| 复合约束    | 多列组合 PRIMARY KEY (包含该类型列)            | 组合主键唯一且非空                       |
| 复合约束    | 多列组合 FOREIGN KEY (包含该类型列)            | 组合外键引用                             |

**修改约束 (ADD / DROP CONSTRAINT) **

| 维度             | 测试点                         | 说明                               |
| ---------------- | ------------------------------ | ---------------------------------- |
| 添加 UNIQUE      | 表中无重复值时添加 UNIQUE 约束 | 成功                               |
| 添加 UNIQUE      | 表中有重复值时添加 UNIQUE 约束 | 应报错                             |
| 添加 PRIMARY KEY | 表中无 NULL 且无重复时添加主键 | 成功                               |
| 添加 PRIMARY KEY | 表中有 NULL 或重复时添加主键   | 应报错                             |
| 添加 FOREIGN KEY | 引用表存在且数据满足引用完整性 | 成功                               |
| 添加 FOREIGN KEY | 引用表不存在或数据不满足完整性 | 应报错                             |
| 添加 CHECK       | 表中所有数据满足 CHECK 条件    | 成功                               |
| 添加 CHECK       | 表中存在数据不满足 CHECK 条件  | 应报错                             |
| 删除 UNIQUE      | 删除 UNIQUE 约束               | 成功，后续可插入重复值             |
| 删除 PRIMARY KEY | 删除主键约束                   | 成功                               |
| 删除 FOREIGN KEY | 删除外键约束                   | 成功                               |
| 删除 CHECK       | 删除 CHECK 约束                | 成功                               |
| 启用/禁用约束    | 禁用约束 (若数据库支持)        | 禁用时约束不生效                   |
| 启用/禁用约束    | 启用约束 (若数据库支持)        | 启用后约束重新生效，已有数据需满足 |

**多个约束/属性添加在同一字段上**

| 维度                                | 测试点                                               | 说明                                                    |
| ----------------------------------- | ---------------------------------------------------- | ------------------------------------------------------- |
| NOT NULL + DEFAULT                  | 同一字段同时添加 NOT NULL 和 DEFAULT (默认值非 NULL) | 插入时不指定该列使用默认值，不能显式插入 NULL           |
| NOT NULL + UNIQUE                   | 同一字段同时添加 NOT NULL 和 UNIQUE                  | 唯一且非空                                              |
| PRIMARY KEY + DEFAULT               | 同一字段同时添加 PRIMARY KEY 和 DEFAULT              | 主键列一般不允许 DEFAULT，视数据库行为                  |
| DEFAULT + CHECK                     | 同一字段同时添加 DEFAULT 和 CHECK                    | 默认值必须满足 CHECK 条件，否则建表失败                 |
| UNIQUE + CHECK                      | 同一字段同时添加 UNIQUE 和 CHECK                     | 同时满足唯一性和 CHECK 条件                             |
| NOT NULL + DEFAULT + UNIQUE + CHECK | 四个约束同时添加在同一字段                           | 验证所有约束同时生效                                    |
| 跨字段组合                          | 同一表内多个字段各有不同约束组合                     | 字段间约束相互影响 (如复合唯一索引、外键引用带约束的列) |

# 二、类型特性补充指引

在生成用例时，要求 AI 根据**类型自身特性**补充用例。将以下问题作为检查清单 : 

## 2.1 数值/精度类 (INT、DECIMAL、FLOAT) 

- 精度与标度边界 (最大/最小位数) 
- 舍入模式 (四舍五入、截断、向零取整、向上/向下取整) 
- 溢出与下溢 (报错或回绕) 
- 特殊值 : NaN、Inf、-Inf、-0
- 除零错误 (整数除零、浮点除零) 
- 取模运算符号规则 (与被除数一致或与除数一致) 
- 算术运算结果的类型提升规则 (INT + BIGINT → BIGINT) 

## 2.2 布尔类 (BOOLEAN、BIT) 

- 真值表示 (TRUE/FALSE，1/0，t/f，yes/no，on/off，'true'/'false' 大小写不敏感) 
- NULL 三值逻辑 (AND/OR/NOT 真值表) 
- 与数值类型的隐式转换 (0→FALSE，非0→TRUE；TRUE→1，FALSE→0) 
- 与字符类型的转换 ('true'→TRUE，'false'→FALSE；前后空格处理) 
- 逻辑操作符短路行为 (AND 左侧 FALSE 不计算右侧；OR 左侧 TRUE 不计算右侧) 
- 比较谓词返回类型 (a = b 返回 BOOLEAN) 
- 布尔列在 GROUP BY 中的分组行为 (NULL 单独一组) 
- 布尔列在 ORDER BY 中的排序 (FALSE < TRUE，NULL 位置取决于数据库) 

## 2.3 字符/文本类 (CHAR、VARCHAR、TEXT、CLOB) 

- 长度边界 (最小0/1，最大定义值，超过截断或报错) 
- 字符集与排序规则 (UTF-8、GBK、Latin1 等，不同编码下字符串比较) 
- 空串 '' 与 NULL 的区别
- 转义字符与特殊字符 (\n, \t, \', \\ 等) 
- 前后空格保留或截断 (CHAR 类型通常空格填充，VARCHAR 保留) 
- 与数值类型的转换 ('123'→123，'12.34'→12.34，'abc'→报错) 
- 与布尔类型的转换 ('true'→TRUE，'false'→FALSE) 
- 字符串连接 (|| 或 CONCAT) 对 NULL 的处理
- LIKE 模式匹配中通配符 (%、_) 及转义符 ESCAPE
- 大小限制与性能边界

## 2.4 时间/日期类 (DATE、TIME、TIMESTAMP、INTERVAL) 

- 时区与夏令时 (带时区类型、会话时区设置) 
- 闰年、月末、跨日计算 (DATEADD、DATEDIFF) 
- 区间合法性 (如 end > start) 
- 格式字符串解析 (隐式转换的格式依赖，如 '2024-01-01' 自动转为 DATE) 
- 日期运算 (date + interval, date - date) 
- 日期截断 (TRUNC、DATE_TRUNC) 
- 时区转换 (AT TIME ZONE) 

## 2.5 向量/数组/复合类 (VECTOR、ARRAY、JSON) 

- 维度一致性 (不同维度向量运算应报错) 
- 零向量、单位向量
- 稀疏表示 (若适用)  : 空稀疏、全稠密
- 距离/相似度计算的数学正确性 (欧氏距离、余弦相似度等) 
- 下标越界访问 (应报错或返回 NULL) 
- 数组元素类型混合 (如 INT 数组中插入 VARCHAR，应报错) 
- 数组与标量运算 ([1,2,3] + 1 → [2,3,4]) 
- JSON 路径提取 (->、->>、#>> 等) 
- JSON 类型转换 (jsonb 与 text 互转) 
- 嵌套深度限制 (超过限制报错) 

## 2.6 结构化类 (UDT、ROW、RECORD) 

- 空结构 (全部属性为 NULL) 
- 深层嵌套 (UDT 中包含 UDT) 
- 属性路径不存在 (访问不存在的字段应报错) 
- 类型不匹配赋值 (UDT 属性类型与赋值值类型不一致) 
- 与其他类型的转换 (如 UDT 转 JSON、ROW 转 RECORD) 
- 结构化类型的比较 (按属性逐一比较，或定义特定比较函数) 
- 结构化类型在 SQL 语句中的使用 (SELECT、WHERE、GROUP BY 等) 
- 结构化类型的默认值 (是否允许，如何定义) 
- 结构化类型的索引支持 (如 UDT 属性上创建索引) 
- 结构化类型的序列化/反序列化 (如存储为 JSON、XML) 

## 2.7 自定义/扩展类型

- 与基础类型的互操作 (隐式/显式转换) 
- 序列化/反序列化 (存储与读取一致性) 
- 跨会话、跨节点一致性 (分布式环境) 
- 比较函数/操作符自定义实现 (如 C 语言扩展)
