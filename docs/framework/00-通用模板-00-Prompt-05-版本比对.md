# 约束与要求

**要求** :
- 结构完整 : 上表**五章 + 文首 + 约定** 须全部出现
- 逻辑内容 : **§1 总览** 每一行 **Cxx** 在 **§3** 有对应细则 ; **§2 动机** 与 **Cxx** 同序
- 编号列表 : **C01～Cn** ; 不用 FR/TD/GS 充当变更号 ( 金样在需求/用例中另引 **Cxx** )
- 文档信息 : 文首标明 A/B 版本、比对范围 ( **仅本产品 DESIGN 差异** , 不含未入设计的纯缺陷修复 )
- 内容详尽 : 每项含**风险**与**迁移/回归提示** ; 矩阵类变更 ( 多子类型 ) 用表格 , 不用大段散文
- 包含内容 : 状态符号 🔄 / 🌓 / ➖ ( 与母版一致 )
- 兼容性 : §4 五类项必填 ; 未知写**待确认** , 不编造迁移脚本存在性
- 精简 : 能用表则表 ; 不与 Oracle/PG/MySQL 竞品对比
- **工作流程** :
  1. **定位仓库** : 在含 `README.md`、`docs/` 的**本项目根目录**工作 ; Agent 模式须**实际执行**终端 , 不得只列命令
  2. **文档落盘** : 按本章「输出格式」写入 `docs/<主题>/` ( 如 `docs/interval/INTERVAL-05-版本比对.md` ) , 替换 `{输出文件}` 占位 ; 须已读同主题设计/需求及 A/B 版本依据
  3. **本阶段产物** : 仅版本比对 Markdown ; **不**生成 `scripts/` SQL 与 `matrix/` 矩阵 ( 回归脚本引用 **Cxx** 与金样 **GS-xxx** 即可 )
  4. **调用项目** ( 按需 ) : 仅当需验证升级后行为时 , 才 `.\run.ps1` / `.\run-connect.ps1` , 报告在 `reports/`

**约束** :
- 文章中的 `,` , `:` , `/` 等均使用英文并且前后各有一个空格 ; `" "` , `()` , `<>` , `** **` 以及行内代码块等在前后各加一个空格,如果配合使用则两个符号之间不用空格隔开,例如 : `**()**` 表示包括括号在内的内容加粗,文章中不要出现任何的中文标点符号
- 对于编号列表,要在文档开始以表格的形式总览,在后续章节中详细说明
- 表格中的编号列居中对齐,文档约定的符号例如 ✔️ / 🔄 等也居中对齐
- 当有编号的时候,在文档最前面的章节要有总览,包含编号、说明,之后在后续章节中详细展开描述
- 缺 A/B 设计依据时不编造 **Cxx** 条目
- 不要把三库竞品差异写进版本比对
