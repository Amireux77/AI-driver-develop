# 测试项目 — 统一编号体系

> **文档版本** : 2.1  
> **编写日期** : 2026-05-22  
> **适用范围** : `测试` 目录下设计 / 可行性 / 需求 / 测试方案 / 用例设计 / 版本比对及通用模板  
> **原则** : 追溯编号**仅保留角色前缀 + 序号** ; 主题由**文档文件名**表达 (例如 `{主题}`) ; 可行性缺口 **G** 仅出现在可行性分析 , 需求侧用 **GR** 承接

---

## 1. 编号一览

| 代码                      | 格式示例                    | 含义                        | 主要出现文档            |
|---------------------------|-----------------------------|-----------------------------|-------------------------|
| **§**                     | §2.1                        | 章节号                      | 全部                    |
| **A (Assume)**            | A1～A4                      | 关键假设                    | 可行性、需求            |
| **Q (Require)**           | Q1～Q7                      | 待确认问题                  | 需求、可行性            |
| **L (Layer)**             | L1～L3                      | 测试分层                    | 需求                    |
| **G (Gap)**               | G01～G15                    | 竞品缺口 ( 可行性发现 )     | **仅** 可行性           |
| **GR (Gap Reuqire)**      | GR-001                      | 缺口需求 ( 与 G 同序 )      | 需求清单                |
| **FR (Function Require)** | FR-001                      | 功能需求                    | 需求、各功能章          |
| **PD (Pending)**          | PD-001                      | 待实现需求 ( Pending )      | 需求清单                |
| **AC (Acceptance Check)** | AC-01                       | 验收项                      | 需求验收标准            |
| **D (Design)**            | D-01                        | 设计项                      | 设计文档                |
| **TD (Test Domain)**      | TD-01                       | 测试域                      | 可行性、测试方案        |
| **XG-…**                  | XG-TYPE-<主题>-TABLE-001 等 | **可执行用例编号** ( 唯一 ) | 用例设计、FitNesse 脚本 |
| **GS (Gold Sample)**      | GS-001                      | 回归金样                    | 需求、用例设计          |
| **C (Change)**            | C01～C06                    | 版本设计变更                | 版本比对                |
| **T (Trans)**             | T01～T13                    | INTERVAL 十三子类变体       | INTERVAL 需求/用例      |
| **P (Priority)**          | P0                          | 优先级 ( 非独立需求 ID )    | 全篇                    |

### 用例编号格式 ( XG )

**统一格式** : `XG-TYPE-<主题缩写>-<域代码>-<三位序号>`

| 主题系列 | 主题缩写 `<主题>` | 示例                          |
|----------|-------------------|-------------------------------|
| 示例主题 | `<主题>`          | `XG-TYPE-<主题>-TABLE-001`    |
| 示例主题 | `<主题>`          | `XG-TYPE-<主题>-VAL-001`      |
| 示例主题 | `<主题>`          | `XG-TYPE-<主题>-SET-001`      |
| 其它类型 | 与类型名一致大写  | `XG-TYPE-<主题>-<域代码>-001` |

- **域代码** : 与《测试方案》中测试域对应 ( 如 `TABLE`、`VAL`、`UPS` ) , 测试域含义写在用例**标题** ( `!4` 行第二段 ) 或小节标题中 , **不**再使用 `TC-` 前缀  
- **废弃编号不重用** ; 脚本与用例设计表内编号一致

## 2. 状态符号 ( 全项目统一 )

| 符号 | 含义                |
|:----:|---------------------|
|  ✔️   | 已实现 , 可测       |
|  ❌   | 未实现              |
|  ⚠️   | 已实现但有已知差异  |
|  🔺  | 部分实现 / 等价路径 |
|  ➖   | 不适用              |
|  🔄  | 版本比对 : 变更     |
|  🌓  | 版本比对 : 部分变更 |
