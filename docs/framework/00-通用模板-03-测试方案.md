## 4. 测试点编号规则

### 4.1 用例编号格式

**格式** : `<前缀>-<类型缩写>-<域代码>-<三位序号>`

**示例** : `XG-TYPE-INTERVAL-BOUND-003`（前缀、类型缩写按项目约定）

| 段       | 填写说明                                       |
| -------- | ---------------------------------------------- |
| 前缀     | 项目统一前缀 , 如 `XG-TYPE`                    |
| 类型缩写 | 与类型名一致 , 大写 , 如 `INTERVAL`、`NUMERIC` |
| 域代码   | 见 **§4.2** , 与 TD 对应                       |
| 序号     | 域内 `001` 起递增 , 废弃编号不重用             |

### 4.2 域代码与 TD 对照

| 域代码    | 对应 TD | 含义                |
| --------- | ------- | ------------------- |
| `DEF`     |         | 类型定义/DDL 声明   |
| `STORE`   |         | 存储规格            |
| `BOUND`   |         | 值域边界/越界       |
| `IN`      |         | 输入格式            |
| `OUT`     |         | 输出格式            |
| `CAST`    |         | 类型转换            |
| `OP`      |         | 运算与比较          |
| `FUNC`    |         | 系统函数            |
| `AGG`     |         | 聚合函数            |
| `TABLE`   |         | 表 DML/DQL          |
| `PROC`    |         | 过程化语言/存储过程 |
| `OBJ`     |         | 索引/约束/周边      |
| `PERF`    |         | 大数据量            |
| `CLIENT`  |         | 客户端/驱动         |
| `UPGRADE` |         | 版本升级            |
| `CROSSDB` |         | 跨库对标            |

### 4.3 金样编号

**格式** : `GS-<三位序号>`（如 `GS-001`）

- 与需求 **§28 金样登记** 一致 , 在《用例设计》中与用例编号互引 ; **不重复维护 SQL**（SQL 仅在用例设计）
- 升级/行为变更类金样优先级为 **P0** , 纳入 **TD-12** 或日常回归

## 6. 测试策略与环境

### 6.1 策略

1. 测试内容不超出需求与设计 ; 方案定域与优先级 , 用例定编号与脚本
2. 有变体时 **P0** 覆盖每变体边界与至少一条主路径 DML ; **P1** 可抽样
3. 边界、升级、输出变更须对应需求 **§28 金样** 或《版本比对》
4. **主测库**为 Pass/Fail 判定依据 ; 对标库仅用于 **TD-13**
5. 缺陷须关联 **§4.1 用例编号** 或 **§4.3 金样编号**（`GS-*`）
