## 3. 编号与统一表头

### 3.1 编号规则

- **编号格式** : `XG-TYPE-<主题缩写>-<域代码>-<三位序号>` ( 与 FitNesse `!4` 行一致 )
- **示例** : `XG-TYPE-INTERVAL-TABLE-001` , `XG-TYPE-INSERT-VAL-001`
- **用例标题** : `!4` 第二段写明测试域/场景 ; 表中「测试点」与之对应
- **域内序号** : 从 `001` 递增 , 废弃编号不重用
- **金样** : `GS-001`～ , 与本节用例互引

### 3.2 序号分配原则

1. 每个 **§4.x** 对应一个测试域 ; 子节 (**§4.x.y**) 对应脚本文件或子主题 , 序号在**域代码**内连续
2. **P0** 须覆盖各变体 (§2) 边界 + 至少一条主路径 DML/运算
3. 矩阵类 (运算/转换) 可一条编号覆盖多组合 , 须在「说明」列列出维度
4. 跨库 (**TD-13**) 可与主测同编号加后缀或单独 `CROSSDB` 段 , 须在「执行库」列区分
